Components

  • With react we describe the UI using components
  • Components are like JavaScript functions
  • A component takes in parameters called props(short for properties) and returns a hierarchy of views to display via the render method
  • The render method takes the description and displays the result
  • render returns a react element which is a light weight description of what to render
  • most React developers use JSX which makes these structures easier to write
  • A component can maintain internal state data (accessed via this.state)
  • When a components state data changes, the rendered markup will be updated by revoking render()
  • We use components to tell React what we want on our screen
  • When our data changes, React will efficiently update and re-render out components
  • Components receive certain input objects and they output description of a user interface
  • We can reuse a single component in multiple user interfaces
  • Components can contain other components
  • You just use a component in your HTML just like it was a normal HTML element
  • React components can hold a private state to hold any data that may change over the lifecycle of the component to hold any data that may change over the lifecycle of the component
  • A react component can be one of two types
    • Function Component
    • Class Component
  • Both types of components can be compared to regular functions when it comes to their contract
    • Both use a set of props and state as their input, and they output what looks like html, but what is really output is a javaScript syntax called jsx
    • The props input is an explicit one, it is similar to the list of attributes an HTML element has
    • The state input is an internal one, and auto reflects the changes in the browser
    • The two input objects have a difference: within a component the state object can be changed while the props object represents fixed values
    • Props are immutable
    • Components can change their internal state, not there properties
  • to remember things components use state
  • React components can have a state by setting this.state om their constructors
  • This state should be considered as private to a React component that it's defined in
